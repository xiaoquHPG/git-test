团队内协作
如何进行团队内协作
首先我的本地库 push到远程库里面去  然后  被他 clone 到他的本地库然后   他push提交给远程库 我pull下来
团队外写作
从我的本地库 fork到 外团队的远程库   外团队 clone下来  然后 修改好后 push自己的远程库  提出pull 请求  我们团队的远程库审核后就可以拿到了

实际操作如何去进行 github的 团队协作呢 首先打开github 登录账号 点右上角的+号  new 库 就可以了  创建好 拿到链接xxxxxxxxxxx
回到自己的 库下面 gitbash       git remote -v查看自己有无远程库    然后 git remote add 别名 链接 就可以了
再次查看 git remote -v就可以了
执行这一系列操作后 如果需要提交那个分支的版本使用 git push 链接 分支名 就可以了

如果在后面发现自己的版本跟团队的版本已经不同步了，可以把远程库内容 pull下来      git pull 链接 分支名 就可以了改变本地库了


如果是一个新人来公司的话首先先down下来 代码在一个新的文件夹下面  git clone 就可以了  git clone 加链接就可以了
如果当初创建这个库的时候 是 public 就不需要 验证就可以down下来了 如果是private 就需要 验证 用户信息才能down下来

这个时候如果我是那个新人 我直接去 pull 某个分支 进入后会显示没有权限       得在创建库的那个账号 点击setting 然后点 account access 邀请我这个 id
后在那个框里面 点击那个 邀请链接 然后去登录 我的 github账号 把链接粘贴在 搜索引擎 然后打开 点击就可以了

如何进行外团队协作呢
首先我们是外团队的账户 然后点击了 你们团队的 这个库查看  点击右上角的 fork插入 然后进行如 编辑提交   点 pull request 提交就可以了  我们团队
接受到以后感觉可以采纳 就点 合并分支就可以了

首先在 当前本地库 创建一个 ssh-keygen -t rsa -C "你的github登录邮箱"
然后去你的C盘目录文件下有一个 ssh的文件 cat 进入后 复制
到github 的setting 添加 ssh  把内容 粘贴到 里面  后头 发现里面的ssh链接可以使用了以后pull push clone 就不需要 登录了直接利用ssh的 免密链接就可以了
